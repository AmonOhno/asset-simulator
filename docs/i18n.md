# 多言語対応（I18N）方針

## 現行サイト

- 英語版ページ `https://www.city.saitama.lg.jp/en/index.html` が存在（確認済み）
- 多くの自治体サイト同様、機械翻訳サービスによる多言語化と推定（**要実測**：
  対応言語一覧・翻訳方式は capture 実行時にヘッダーのリンクから確認する）

## モダナイズ版

- **i18next / react-i18next** による構造化リソース方式（`app/src/i18n/`）
  - `ja.json`（既定） / `en.json`
  - 機械翻訳リンク方式と違い、URL を変えずに UI 文言を完全制御できる
- 言語切替（F03）で `document.documentElement.lang` も更新
  （スクリーンリーダーの発音辞書切替に必要）
- 日付は ISO 8601（`<time dateTime>`）で保持し、表示のロケール整形は今後
  `Intl.DateTimeFormat` へ移行可能
- コンテンツ（新着情報等）は DB 側が単一言語のため、多言語化する場合は
  `news_translations` テーブル追加（`news_id + locale` ユニーク）が次のステップ

## 拡張時の指針

- 対応言語追加は `app/src/i18n/<locale>.json` を追加し `index.ts` に登録
- 中国語（簡体/繁体）・韓国語・やさしい日本語は自治体サイトで需要が高い
  （現行サイトの対応言語一覧の実測後に優先度を決める）
