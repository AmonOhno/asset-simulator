# 現行サイトのテックスタック調査

対象: https://www.city.saitama.lg.jp/

## 調査状況

本セッションでは egress ポリシーにより現行サイトへ直接アクセスできなかったため、
**確認済み事実**と**未確認（要実測）** を明確に分けて記載する。
実測は `capture/capture.mjs`（レスポンスヘッダー・meta generator・HTML 構造を保存）
をアクセス可能な環境で実行して行う。

## 確認済み事実（検索エンジンのインデックス経由）

| 項目 | 内容 | 根拠 |
|------|------|------|
| ドメイン | `www.city.saitama.lg.jp`（地方公共団体ドメイン lg.jp） | 検索結果 |
| URL 構造 | `/001/`〜`/006/` の数値ディレクトリ + `index.html`、記事は `pNNNNNN.html` 形式 | 検索結果の実 URL |
| ページ形式 | `.html` 静的ファイル（拡張子ベース） | 検索結果の実 URL |
| 記事 ID 規則 | `p` + 6 桁数値（例: `p111295.html`, `p046091.html`） | 検索結果の実 URL |
| 添付ファイル規則 | `pNNNNNN_d/fil/<ファイル名>` | 検索結果の実 URL |
| 別動線 | 携帯サイト `/mobile/`、英語版 `/en/`、`/jp/` | 検索結果の実 URL |
| 区ごとのサブサイト | `/nishi/` など区名ローマ字のディレクトリ | 検索結果の実 URL |
| 更新体制 | 各課がページを作成し広報課が全体構成を管理 | 市 FAQ（p111295） |

## 推定（未確認・要実測）

- **CMS**: 静的 HTML 出力型の自治体向け CMS と推定される。
  `pNNNNNN.html` + `_d/fil/` という添付ファイル規則は国内自治体 CMS に
  典型的なパターンだが、**製品名は未確認**。実測時に
  `meta[name=generator]`・レスポンスヘッダー・HTML コメントで特定する。
- **フロントエンド**: ビルド済み JS フレームワーク（React 等）ではなく、
  テンプレート出力の HTML + 素の JS/jQuery 系と推定。**要実測**。
- **レスポンシブ対応**: `/mobile/`（携帯サイト）が別に存在することから、
  本体はレスポンシブでない可能性がある。**要実測**
  （capture.mjs は非レスポンシブ検出時に倍率調整キャプチャを自動取得する）。
- **サイト内検索**: 外部検索エンジン（カスタム検索）利用の可能性。**要実測**。
- **アクセシビリティ機能**: 文字サイズ変更・色合い変更・音声読み上げ・
  多言語（機械翻訳）は自治体サイトの標準装備であり検索結果の説明文にも
  言及があるが、実装方式は**要実測**。

## モダナイズ版のテックスタック（本リポジトリ）

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React 19 + TypeScript + Vite 6（モバイルファースト SPA） |
| ルーティング | react-router-dom 7 |
| I18N | i18next / react-i18next（ja / en） |
| スタイル | CSS Custom Properties によるデザイントークン（フレームワーク非依存） |
| バックエンド | Supabase（PostgreSQL + RLS + Edge Functions） |
| 認証 | パスキー（WebAuthn, @simplewebauthn/browser + server） |
| ローカル環境 | Supabase CLI + Docker（`supabase start`） |
| キャプチャ/検証 | Playwright（capture/） |
