# アクセシビリティ（A11Y）方針

自治体サイトは JIS X 8341-3:2016（WCAG 2.0/2.1 相当）レベル AA 準拠が
事実上の要件。モダナイズ版で実装した内容と根拠を示す。

## 実装済み

| 項目 | 実装 | WCAG 達成基準 |
|------|------|--------------|
| スキップリンク | `App.tsx`（F02）フォーカス時のみ表示 | 2.4.1 ブロックスキップ |
| ランドマーク | `header` / `nav` / `main` / `footer` + `aria-label` | 1.3.1 / 4.1.2 |
| 見出し階層 | h1（ページ）→ h2（セクション）の直列構造 | 1.3.1 |
| 文字サイズ切替 | F04: 16px ⇄ 20px（rem ベースで全体スケール） | 1.4.4 テキストのサイズ変更 |
| ハイコントラストモード | F04: `data-contrast="high"` でトークン切替 | 1.4.3 / 1.4.6 |
| コントラスト比 | 本文 17.6:1、リンク 6.9:1（AA 以上） | 1.4.3 |
| フォーカス表示 | `:focus-visible` に 3px アウトライン | 2.4.7 |
| タップターゲット | 最小 44×44px（`--tap-target-min`） | 2.5.8 |
| フォームラベル | 検索・ログインとも `label` を関連付け | 3.3.2 / 4.1.2 |
| 動的更新の通知 | 検索結果 `role="status"`、認証状態 `aria-live="polite"` | 4.1.3 |
| 言語宣言 | `html lang` を言語切替に追随して更新 | 3.1.1 |
| 状態の表現 | 切替ボタンに `aria-pressed` | 4.1.2 |
| 緊急情報 | 色だけに依存せずラベル文字を併記 | 1.4.1 |

## 現行サイトの A11Y 機能（踏襲対象）

- 文字サイズ変更 → F04 で踏襲
- 色合い（コントラスト）変更 → F04 で踏襲
- 音声読み上げ → **未実装**。モダン実装ではブラウザ/OS のスクリーンリーダー
  （VoiceOver / TalkBack）が標準のため、セマンティック HTML での対応を優先した。
  専用読み上げウィジェットが必要かは要件次第（missing-info.md）。
- ふりがな表示 → **未実装**（現行サイトでの提供有無も要実測）

## 検証方法

- axe DevTools / Lighthouse Accessibility を CI に組み込むのが次のステップ
- キーボードのみでの全操作確認（Tab 順序はソース順 = 視覚順）
- スクリーンリーダーでのランドマークジャンプ確認
